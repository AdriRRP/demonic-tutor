# Cast Flash Enchantment In Beginning Of Combat Window

## Summary

Allow a supported enchantment card with explicit `OpenPriorityWindow` permission to be cast in `BeginningOfCombat`.

## Scope

- reuse the minimal `Flash`-like casting rule already supported on card faces
- prove the behavior in the first combat priority window for a noncreature permanent

## Out Of Scope

- richer combat interaction beyond the supported stack corridor
