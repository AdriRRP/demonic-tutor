# Slice Proposal - Support Cast From Graveyard On Explicit Card Profile

## Goal

Allow one explicit supported card profile to be cast from its controller's graveyard.

## Why This Slice

This is a high-value way to approach graveyard playability without claiming broad flashback or escape support.

## Scope

- one explicit casting-permission profile that allows cast from graveyard
- normal cost payment and stack behavior
- card leaves graveyard and follows normal resolution

## Out of Scope

- generic flashback, escape, jump-start, disturb, or aftermath frameworks
- alternative graveyard costs
- exiling the card instead of normal destination unless explicitly supported later

## Notes

- keep this as an explicit profile, not a generic rules engine
- it should come after graveyard targeting and movement are already stable
