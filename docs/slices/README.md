# Slice Docs

This directory keeps the repository's slice history and planning backlog grouped to reduce cognitive load.

## Structure

- `implemented/`
  implemented and historical slices, grouped by capability
- `proposals/`
  live planning backlog, grouped by implementation wave

`proposals/` may be intentionally empty between release horizons.

## Reading Guidance

- use canonical docs for current repository truth
- use `implemented/` to understand how a capability entered or evolved
- use `proposals/` to understand the current planned backlog

Start here:

- `implemented/README.md`
- `proposals/README.md`
