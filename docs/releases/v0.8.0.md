# Release Notes - v0.8.0

## Summary

`v0.8.0` is the release where DemonicTutor becomes an honestly playable curated limited shell instead of only a UI-ready engine contract.

The main theme of this release is:

- freeze one explicit playable subset
- validate one deliberately constrained curated environment end to end
- harden the public runtime contract for a real first client shell
- close the `0.8.0` proposal horizon honestly

---

## Highlights

- The project now exposes `playable_subset_version = v1` and rejects curated card loads that exceed the authored supported-profile catalog.
- The first curated environment is covered by real golden matchups across white-blue tempo, black-red value/sacrifice, and green-white counters.
- Seeded setup, rematch, replay-ready public logs, and deterministic public command envelopes are now strong enough to support a real first best-of-one client shell.
- Final elite-audit passes hardened observable event ordering, visible zone-move semantics, replay/public-surface consistency, and several hot read paths.

---

## Why It Matters

`v0.7.0` made the engine honestly ready for a frontend.

`v0.8.0` turns that honest client contract into a deliberately constrained but actually shippable playable shell:

- the supported card pool is now frozen and explicit
- unsupported curated cards are rejected instead of being left to undefined behavior
- the intended archetypes are validated through golden gameplay, not only by individual slices
- the public runtime surface is much tighter for replay, animation, and session bootstrapping

This still does not claim full Magic support.

It does mean the repository now has one explicit, honest answer to the question:

"What can DemonicTutor play right now?"

---

## Included In This Release

- curated-set capability freeze and supported-profile rejection
- first frozen `playable_subset_version = v1`
- seeded public setup, rematch, and replay/session helpers for the constrained product shell
- golden matchups for white-blue tempo, black-red value/sacrifice, and green-white counters
- release-blocking hardening for observable zone changes, replay/public envelope consistency, lifecycle/session startup, and public-surface hot paths

## Upgrade Note

This release is the first intentionally frozen playable contract for a curated limited shell.

The most important practical shift is that downstream UI or product work can now rely on:

- one explicit playable subset version
- one curated capability matrix and deck-construction baseline
- deterministic seeded session bootstrapping
- a replay-friendly public runtime contract that has already gone through multiple hardening waves

Further expansion after `v0.8.0` should now happen as a new explicit horizon, not as hidden drift inside the released shell.
