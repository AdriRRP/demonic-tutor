# Release Notes - v0.7.0

## Summary

`v0.7.0` is the release where DemonicTutor becomes honestly ready for the first real UI.

The main theme of this release is:

- stabilize the public gameplay contract
- broaden common card-pattern support
- harden ownership, stack, and event semantics
- close the last release-blocking coherence gaps before frontend work

---

## Highlights

- The engine now exposes a stable public game snapshot, legal-action surface, visible choice requests, and deterministic public command results for UI clients.
- Supported player-choice corridors now include modal `choose one`, `you may`, `loot`, `rummage`, `scry 1`, and `surveil 1`.
- The first attachment and static-effect baseline is live: creature Auras, attached stat bonuses, pacifism-style restrictions, and a controller anthem.
- The subset now supports key board-control utility patterns such as `tap`, `untap`, and `can't block this turn`.
- Tokens and graveyard value patterns are broader and more useful for limited-style play, including multi-token creation, spell recursion, bounded graveyard casting, and end-step spell recursion.

---

## Why It Matters

`v0.6.0` made the engine much more playable.

`v0.7.0` turns that broader subset into something a frontend can build against honestly:

- the UI no longer needs to invent legal actions or prompt semantics
- the observable event stream is much more truthful and replay-friendly
- owner/controller-sensitive zone movement is no longer papered over in the supported subset
- the supported card pool now covers enough common patterns to stop feeling like a toy contract

This still does not claim full Magic support.

It does mean the project now has a coherent, explicit, UI-ready gameplay foundation.

---

## Included In This Release

- public gameplay projection, legal-action queries, and choice-request contract
- modal choices, optional decisions, loot/rummage, scry, and surveil
- creature Auras, aura-based stat changes, pacifism-style restrictions, and controller anthem support
- tap / untap / cannot-block utility effects and bounded defender-style attack restrictions
- keyworded tokens, multi-token creation, broader graveyard spell value, and end-step spell recursion
- release-blocking hardening for ownership, attachments, SBA coherence, public legality queries, and stack event ordering

## Upgrade Note

This release is the intended starting point for frontend work.

The most important practical shift is that UI code can now rely on one explicit runtime contract for:

- visible state
- legal actions
- pending choices
- deterministic emitted events

Remaining gameplay expansion is now deliberately pushed into the `0.8.0` backlog instead of being left as hidden uncertainty inside the `0.7.0` surface.
