# Release Notes — v0.5.0

## Summary

`v0.5.0` closes the broad runtime architecture refactor wave and leaves the current engine on a much more compact, stable, and human-readable foundation.

The main theme of this release is:

- compact internal runtime representations
- readable public boundaries
- lower cognitive load for humans working in the repository

---

## Highlights

- Runtime identity now prefers numeric-core ids, player indices, and player-owned card handles internally.
- Player-owned card runtime is now canonically handle-first instead of public-id-first.
- Stack objects are slimmer and rely more on internal target and source references.
- Ordered zones now have much tighter visible indexing and ordered-removal behavior.
- The aggregate card-location index is maintained transactionally during supported moves.
- The first supported non-mana activated ability corridor now rides the same priority-and-stack model as other stack interactions.

---

## Why It Matters

This release does not mainly add breadth to the Magic subset.

It improves the quality of the engine underneath that subset:

- less hot-path public-id churn
- less duplicated spell metadata in flight
- better runtime locality
- clearer ownership boundaries inside the `Game` aggregate
- more durable foundations for future slices

In practical terms, the current supported gameplay is now implemented on top of a runtime model that is easier to evolve without reintroducing storage drift or identity confusion.

## Included In This Release

- broader targeted effect support already landed during the runtime audit waves
- explicit life gain and life loss spell effects
- destroy and exile target creature support
- exile target card from graveyard support
- broader actor-relative and combat-relative targeting support
- temporary pump effects that can change combat outcomes
- `Haste`, `Vigilance`, `Trample`, and `First strike` support in the current subset
- supported non-mana activated abilities through the stack

- code paths align better with the aggregate model
