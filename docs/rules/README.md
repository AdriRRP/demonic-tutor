# Rules Documentation

This directory contains DemonicTutor-owned materials for working with Magic Comprehensive Rules.

The project should not treat the Comprehensive Rules as a literal implementation backlog.

Instead, this directory exists to support a hybrid workflow:

- `rules-map.md` links supported repository behavior to rule sections
- `notes/` contains focused repository-owned summaries for rule areas currently modeled
- `/features/` contains behavior-facing specifications derived from those references

Normative precedence remains:

1. implemented code
2. accepted ADRs
3. canonical repository documentation
4. external rule references

The repository may reference the Comprehensive Rules, but should avoid copying large sections verbatim into project docs.
